**After any non-trivial change, run in order: clippy, then the CRAP gate, then format.** Each step gates the next; formatting is always last.

**1. Clippy.** Use `cargo clippy --workspace --all-targets`, not `cargo check` — the workspace opts into `clippy::pedantic`, so plain `check` is too lenient and gives a misleading "clean" answer. Confirm zero warnings before claiming code is clean. `cargo clippy --fix --allow-dirty` auto-applies most fixes; finish the rest manually. Real correctness flags (`cast_possible_truncation`, `cast_sign_loss`, etc.) warrant a fix; cosmetic pedantic flags on intentional code (e.g. `match_same_arms` on a documentation-style mapping table) can be `#[allow(...)]`'d with a justification comment.

**2. CRAP gate.** Run `just crap-ci`. CRAP = `cyclomatic² × (1 − coverage)³ + cyclomatic`; threshold is 30 (`.cargo-crap.toml`). It's slow because it instruments the suite under `llvm-cov`. It fails two ways — a baselined entry's CRAP regressed, or a function is newly above 30. Either is a real signal: fix by **adding test coverage** or **reducing cyclomatic complexity** (split big functions into small single-purpose helpers — the orchestrator stays untested but its branch count drops). Don't blindly run `just crap-baseline` to silence the gate — that bakes in complexity debt as accepted. Refresh the baseline only for genuinely grandfathered pre-existing entries, and say so in the commit message.

**Per-function coverage floor: every function must reach ≥90% line coverage** (the CRAP report's coverage column is per-function). Cover anything below it before moving on — extract untestable I/O so the logic is reachable, don't lower the bar. Typical gaps that keep a function under the bar: an `ok_or_else` "missing in response" arm (mock a 200 with the field absent), enum `From` impls that only exercise one variant, and `is_retryable`/`map_error` error arms (construct the error directly — `ResponseValue::new(...)`, a connection-refused `reqwest::Error`, a KMS `SignerSource`).

**Run `just coverage` to check the floor** — it prints per-file line coverage and a per-function distribution histogram, and exits non-zero listing any function below 90%. **GOTCHA: cargo-crap's `coverage` field is a PERCENTAGE (0–100), not a fraction (0–1).** When querying `tmp/crap-now.json` by hand, the floor predicate is `select(.coverage < 90)` — `select(.coverage < 0.90)` silently tests 0.9% and reports everything as passing. (This bit us once: an `Amount::to_decimal` sitting at 50% slipped through a `< 0.60` check back when the floor was 60.) Trust `just coverage`, not an ad-hoc jq threshold.

**3. Format.** Once clippy and CRAP are both green, run `cargo fmt --all` as the very last step before staging (default `rustfmt`, no `rustfmt.toml`). The generated module is declared `#[rustfmt::skip] mod generated;`, so `cargo fmt` leaves `src/generated.rs` exactly as `cargo xtask gen` wrote it. Do NOT run `cargo fmt` while iterating — it churns the diff and obscures what's changing under clippy/CRAP feedback. If it touches files you didn't edit, that's drift from elsewhere; commit those under their own focused message (e.g. `workspace: cargo fmt --all`), don't bundle with logic changes. This overrides §3's "don't touch adjacent formatting" for `.rs` files: `rustfmt` output is canonical.

## Time must be injectable in tests

**Never read the clock through a non-mockable source in non-test code.** Banned in `src/` (outside `#[cfg(test)]`): `std::time::Instant::now()`, `std::time::SystemTime::now()`, and any wall-clock read that a test can't control. Time logic (timeouts, refresh margins, expiry, backoff) must be driven by a provider whose "now" a test can advance deterministically.

**Use `tokio::time::Instant` (not `std::time::Instant`).** `tokio::time::Instant::now()` respects `#[tokio::test(start_paused = true)]` + `tokio::time::advance(..)`, so elapsed-time logic is tested without sleeping or flaking. `std::time::Instant::now()` cannot be paused/advanced and is therefore banned. This is the rule behind `auth.rs`'s `TokenManager`: refresh timing runs on `tokio::time::Instant`; the *only* wall-clock read is a single documented `chrono::Utc::now()` at construction to anchor the JWT `exp`, with "now" reconstructed as `anchor + tokio_elapsed` so the whole thing stays advanceable in tests. Any new time-dependent code must follow the same shape — pick a provider you can inject, and prove it with a `start_paused` test that advances the clock.
